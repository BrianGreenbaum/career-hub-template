# Networking

Professional relationships outside of active job search. One file per person.

## How it works

```
Networking/
├── README.md
├── {person-slug}.md   # One file per contact
└── events/            # Event notes and upcoming events (optional)
```

## Contact file shape

Each contact file typically includes:

- Relationship context (how you know them)
- Role and company
- Location
- Conversation history with dates (most recent first)
- Sidebar projects or things they're working on
- Threads to follow up on

## What goes here vs. Search/

- **Networking/** — People you talk to for industry context, advice, informational interviews, or general professional relationships.
- **Search/{year}/companies/[company]/contact-*.md** — People tied to an active job opportunity.
- **Search/{year}/networking/** — External recruiters and agency contacts.

When a networking contact turns into a job opportunity, create the company folder in `Search/` and link to the `Networking/` file from there.
