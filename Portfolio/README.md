# Portfolio

Case studies, presentation decks, and portfolio assets — the materials you use in interviews and on your portfolio site.

## How it works

```
Portfolio/
├── README.md
└── case-studies/
    └── {project-name}/
        ├── outline.md      # Slide-by-slide presentation outline
        ├── dossier.md      # Companion reference with talking points + Q&A
        └── raw-notes.md    # Interview transcript and running capture
```

## Building a case study

Run `/case-study`. It will:

1. Ask what project to build the case study for (pulls existing summaries from `Work-History/` if any).
2. Ask about your target audience (design team? VP of product? hiring panel?).
3. Let you ramble about the project, then ask targeted follow-ups.
4. Propose a section structure for your approval.
5. Generate a slide-by-slide outline, a companion dossier, and the raw interview notes.

The dossier includes anticipated interview questions and talking points — useful even if you never build the actual slides.

## Other portfolio assets

This folder is also a reasonable place to keep your portfolio site content, downloadable resume PDFs, and any static design assets you want under version control.
