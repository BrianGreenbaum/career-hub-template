# Weekly

Professional update log. One entry per ISO week covering work, job search, networking, and personal energy.

## How it works

Entries are filed as `{YYYY}-W{NN}.md` (e.g., `2026-W15.md`). Use the `/update` skill to capture an entry — it scans recent git activity, TODOs, and pipeline movement before interviewing you for the gaps.

## Why bother

- Captures context that's easy to forget later (what happened, how it felt, what you were thinking)
- Gives you something to look back on during interview prep or promotion conversations

If you didn't opt into weekly journaling during `/onboard`, you can still use `/update` any time you want to capture a log.
